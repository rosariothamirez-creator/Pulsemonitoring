# Hardware

## Overview

The hardware subsystem is responsible for acquiring vibration signals and converting them into digital data suitable for software processing.

The system was designed to provide a portable, low-cost and scalable vibration monitoring solution.

---

## Main Functions

- Vibration acquisition
- Signal conditioning
- Analog-to-digital conversion
- Local data storage
- Data transfer to the software subsystem

---

## Hardware Architecture

Accelerometer
↓
ADC
↓
Raspberry Pi
↓
CSV File

---

## Main Components

- Accelerometer
- Analog-to-Digital Converter (ADC)
- Raspberry Pi
- Power Supply
- 3D Printed Enclosure

---

## Design Objectives

- Low-cost implementation
- Portability
- Ease of assembly
- Modularity
- Open-source compatibility
