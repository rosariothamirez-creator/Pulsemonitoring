# Processed Data (Reference Analytical Outputs)

## Overview

This folder contains the static visual outputs generated by applying
PULSE's signal-processing pipeline (`geonode_analyzer.py`) to real events
recorded from the prototype, stored as image files rather than raw
numerical exports.

> **Architectural note:** `geonode_analyzer.py` computes the
> acceleration-to-velocity integration, drift removal, and FFT in memory
> at request time — it does not write intermediate numerical CSVs to disk.
> The files in this folder are therefore the rendered *reports* (plots),
> not raw processed data tables. They let a reviewer inspect the pipeline's
> output visually without running the Python application.

## Contents

### Environmental module reports
* `grafico_np2074.jpg` — Tri-orthogonal PVS analysis under NP 2074:2015
  (Standard/"Corrente" structure class). Shows the 3-axis velocity traces,
  the resultant PVS curve, the applicable limit line, and the FFT-based
  risk-zone spectrum.
* `grafico_din4150.jpg` — PPV analysis under DIN 4150-3:2016 (Sensitive
  structure class).
* `grafico_bs5228.jpg` — PPV analysis under the BS 5228-2 / ISO 2631-2
  fixed-threshold category (impulsive/detonation reference limit).

### Seismic module report
* `grafico_sismograma.jpg` — Time-domain seismogram (PGV, vertical axis)
  and its FFT amplitude spectrum, with the resulting Modified Mercalli
  Intensity classification.


## Important caveat on the measured values

All reporting images were generated from real test events captured by the prototype. However, these impacts were induced **very close to the sensor** (e.g., mechanical impacts centimeters away from the sensor). 

Consequently, this generated extreme Peak Particle Velocities (PPV) far exceeding regulatory boundaries (e.g., 129–156 mm/s vs. the standard 1.5–50 mm/s limits). These outputs strictly validate the algorithmic integrity of the integration, detrending, and thresholding logic; they do not represent macroscopic real-world attenuation scenarios. See `docs/validation.md` for the full technical discussion.

See `docs/software.md` for the exact integration code and `docs/standards.md` for how the dominant frequency is applied in the threshold lookup matrix.
